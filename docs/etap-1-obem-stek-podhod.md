# Этап 1: BI-дашборд — объём, стек и подход

Детализация первого этапа для Meat_coin: что входит в работу, какие инструменты используются, как будет реализовано.

---

## 1. Цель этапа

Получить **веб-дашборд**, который заменяет текущий процесс:

- управляющие вручную вносят цифры из iiko и 1С в Excel;
- фин. аналитик по формулам строит отчёты и картинки.

**Результат:** данные автоматически забираются из iiko (факт) и из 1С:БитФинанс (план), метрики считаются в системе, дашборд отображает графики и таблицы с учётом прав доступа (кто какие рестораны видит). Обновление — по расписанию (например 1 раз в сутки) или по кнопке/онлайн, в зависимости от возможностей источников данных.

---

## 2. Подзадачи (разбивка работ)

| № | Подзадача | Описание |
|---|-----------|----------|
| 1 | **Доступ к данным** | Уточнение формата и способа получения данных: iiko (API, выгрузки, периодичность), 1С:БитФинанс (выгрузки, структура файлов). Получение тестового доступа/примеров. |
| 2 | **Хранение и загрузка** | Настройка БД (PostgreSQL), таблицы под факт/план и справочники. Скрипты/сервисы загрузки из iiko и из файлов 1С. Расписание обновления (cron/планировщик). |
| 3 | **Расчёт метрик** | Перенос бизнес-логики и формул из текущего Excel в код. Согласование с вами расчётов и кейсов (1–2 итерации). |
| 4 | **API для дашборда** | Бэкенд-сервис: отдача уже посчитанных данных по выбранному ресторану/периоду/срезу. Учёт прав: пользователь видит только разрешённые рестораны. |
| 5 | **Веб-интерфейс (дашборд)** | Страница с графиками и таблицами, фильтры по периоду и ресторану. Возможность смотреть один ресторан или свод по всем. Адаптив для работы с планшета/ноутбука. |
| 6 | **Права доступа** | Регистрация/вход пользователей (~20), привязка ролей: «все рестораны» / «только свои». Разграничение в API и в интерфейсе. |
| 7 | **Деплой и эксплуатация** | Развёртывание на вашем сервере или выделенном хостинге (Docker). Инструкция по обновлению и расписанию. |
| 8 | **Приёмка** | Проверка на ваших данных, 1–2 итерации правок по замечаниям. |

---

## 3. Инструменты и стек

Основа — **своя система**, без зависимости от подписок на Power BI или других закрытых BI-платформ.

| Компонент | Технология | Комментарий |
|-----------|------------|-------------|
| **База данных** | PostgreSQL | Надёжное хранение, удобные выборки и агрегаты под отчёты. |
| **Бэкенд и интеграции** | Node.js (Express) или Python | Загрузка из iiko и из файлов 1С, расчёт метрик, REST API для дашборда. У меня большой опыт с Node.js; при необходимости — Python для скриптов обработки данных. |
| **Фронт дашборда** | React + Recharts / Apache ECharts | Современный веб-интерфейс, графики и таблицы, быстрые фильтры. Внешне — уровень Power BI, но под вашим контролем. |
| **Инфраструктура** | Docker, nginx, Linux | Контейнеризация сервисов, развёртывание на одном сервере. При росте — возможность масштабирования. |
| **Планировщик** | Cron или встроенный планировщик в приложении | Обновление данных 1 раз в сутки (или с другой периодичностью по согласованию). |

**Google Sheets / Excel:** могу использовать как вспомогательный слой (например экспорт выгрузки для проверки или быстрый прототип), но основой продакшена будет своя БД и веб-дашборд — так вы не зависите от внешних платформ и подписок.

---

## 4. Что не входит в первый этап (отдельные этапы)

- **Глубокая автоматическая интеграция с 1С «из коробки»** (если у 1С:БитФинанс нет простого API) — при необходимости проектируется отдельно после уточнения возможностей 1С.
- **Второй и последующие отчёты/дашборды** — отдельными этапами по согласованию ТЗ и оценки.
- **Развитие IT-инфраструктуры и упаковка франшизы** — отдельный блок работ; готов обсуждать после запуска первого дашборда.
