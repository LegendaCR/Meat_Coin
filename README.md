## Первый этап: BI-дашборд

**Цель:** заменить ручной Excel на веб-дашборд с автоматическим сбором данных и гибкими правами доступа.

| Что делаем | Описание |
|------------|----------|
| **Данные** | Факт — из iiko (API или выгрузки), план — из 1С:БитФинанс (выгрузки/согласованный формат). |
| **Хранение и расчёты** | Своя БД, перенос формул и метрик из текущей таблицы, обновление по расписанию (например 1 раз в сутки) или по запросу. |
| **Веб-дашборд** | Графики и таблицы, фильтры по периоду и ресторану. Вид «как в Power BI», но своя система — без подписки и привязки к Microsoft. |
| **Права доступа** | До ~20 пользователей: кто-то видит все рестораны, кто-то только свои; в одном дашборде — выбор «один ресторан» или «все». |
| **Результат** | Управляющие и фин. директор видят актуальные отчёты без ручного переноса из iiko и 1С; собственник получает отчёт по запросу на основе тех же данных. |

Детальная разбивка этапа, оценка и используемый стек — в документе **[Этап 1: объём, стек и подход](./docs/etap-1-obem-stek-podhod.md)**.

## Стек и как будет писаться

Использую проверенный стек, на котором уже поднимал похожие системы:

| Слой | Технологии | Зачем |
|------|------------|--------|
| **Данные** | PostgreSQL | Хранение факта, плана, агрегатов; надёжность и гибкие запросы. |
| **Интеграции** | Node.js (или Python) | Выгрузка из iiko, загрузка плана из 1С/файлов, расчёт метрик, API для дашборда. |
| **Дашборд** | React + библиотека графиков (Recharts / ECharts) | Удобный веб-интерфейс, графики и таблицы, фильтры, адаптив. |
| **Инфраструктура** | Docker, nginx, Linux | Развёртывание у вас или на выделенном сервере; вы не зависите от чужих SaaS. |

**Принцип:** своя кодовая база и своя инфраструктура — все доработки, новые отчёты и расширение в IT (в т.ч. упаковка франшизы) делаются в единой экосистеме, без «зашивания» в закрытые платформы.

Подробнее по этапам разработки и инструментам — в **[Этап 1: объём, стек и подход](./docs/etap-1-obem-stek-podhod.md)**.
